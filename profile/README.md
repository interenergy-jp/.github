# Interenergy GitHub Organization

インターエナジーのソフトウェア、設計データおよび関連ドキュメントを管理するGitHub Organizationです。

## 基本方針

 リポジトリは原則として **Private** で作成します

## リポジトリの作成

リポジトリの作成はOrganization管理者が行います。

新しいリポジトリが必要な場合は、以下の情報を添えて管理者へ依頼してください。

* PJ番号
* プロジェクト名
* 顧客案件または社内案件の区分
* 希望するリポジトリ名
* 利用するメンバーまたはTeam
* 管理対象

  * ソフトウェア
  * 組み込みファームウェア
  * 回路・基板設計
  * ドキュメント
  * その他
* 既存の社内Gitから移行するか、新規作成するか

## リポジトリ名

原則として、次の形式を使用します。

```text
PJ番号上3文字（または２文字）-年度2桁-PJCode-内容
```

例： PJ番号　SAE-26001-00

```text
sae-26-repeater-fw
sae-26-repeater-gui
```

複数の構成要素がある場合は、それぞれの役割が分かる名前にしてください。

リポジトリ名には、原則として次の形式を使用します。

* 半角英数字
* 小文字
* 単語の区切りはハイフン
* 略称は社内で意味が共有されているものに限定
* IE以外の会社名、製品名を直接入れない

## ブランチ運用

通常の開発では、デフォルトブランチを直接変更せず、作業ブランチからPull Requestを作成します。下記に具体例を示しますが、ブランチ名は自由に作成してもらって構いません。PRを挙げ、デフォルトブランチに取り込まれると自動で作業ブランチは削除されます。

作業ブランチ名の例：

```text
feature/作業名
edit/作業名
```

基本的な作業手順：

```bash
git switch master
git pull origin master
git switch -c feature/作業内容
```

変更後、作業ブランチをPushします。

```bash
git add .
git commit -m "変更内容"
git push -u origin feature/作業内容
```

GitHub上でPull Requestを作成したら、レビューおよび必要な確認を経てマージされます。

## デフォルトブランチ

既存プロジェクトでは、デフォルトブランチとして `master` を使用している場合があります。

新規プロジェクトで `main` を使用する場合は、プロジェクト内で表記を統一してください。

以下の操作は原則として禁止します。

* デフォルトブランチへの直接Push
* Force Push
* レビューを経ない変更のマージ
* 履歴を書き換える操作
* 管理者への確認なしでのデフォルトブランチ変更

## Pull Request

Pull Requestには、少なくとも以下を記載してください。

* 変更内容概要
* 動作確認結果
* 補足など

記載例：

```markdown
## 変更内容

UART通信のタイムアウト処理を追加しました。

## 動作確認

正常通信を確認 - ケーブル切断後に自動復旧することを確認
```

## レビュー

レビューでは、主に以下を確認します。

* 要求仕様を満たしているか
* 既存機能への影響がないか
* 可読性や保守性に問題がないか
* エラー処理が適切か
* セキュリティ上の問題がないか
* 必要な動作確認が行われているか
* 機密情報が誤って含まれていないか

指摘事項への対応が完了し、未解決の会話がないことを確認してからマージしてください。

## コミット

コミットメッセージは、変更内容が分かるように記載してください。

良い例：

```text
UART受信タイムアウト処理を追加
ログイン失敗時のエラー表示を修正
データベース接続処理を共通化
セットアップ手順を更新
```

避ける例：

```text
修正
更新
test
work
一旦保存
```

可能な範囲で、1つのコミットには関連する変更だけを含めてください。

## 既存の社内Gitからの移行

社内Gitを `origin` としている既存リポジトリは、GitHubへの移行完了後に接続先を変更します。

```bash
git remote -v
git remote set-url origin git@github.com:interenergy-jp/リポジトリ名.git
git fetch origin --prune
git branch -vv
```

未コミットの変更は、`origin` を変更しても失われません。

接続先の変更後に、通常どおりコミットしてGitHubへPushできます。

作業中のブランチを初めてGitHubへPushする場合は、以下を実行します。

```bash
git push -u origin ブランチ名
```

移行後は、旧社内GitへPushしないでください。

## 問い合わせ

以下の場合は、Organization管理者またはプロジェクト責任者へ連絡してください。

* 新しいリポジトリを作成したい
* Teamやメンバーの権限を変更したい
* デフォルトブランチやルールセットを変更したい
* 社内GitからGitHubへ移行したい
* 機密情報を誤って登録した
* リポジトリをアーカイブまたは削除したい
* GitHub Actionsや外部サービスとの連携を追加したい
